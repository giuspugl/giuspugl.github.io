def bin_gains(tau, fs,obs,  g) : 
    sizech = pl.int_( tau *fs)  ; nch =pl.int_(  obs *fs*3600  /sizech) 
    gb = pl.zeros(nch)
    for i in range (nch): 
        gb[i] =g[slice(i*sizech, (i+1)*sizech)] .mean()
    return gb

Outline¶

several improvements have been implemented in the gain drifter module in TOAST :

  1. Making drifts more realistic. I aim at reducing the typical time scale scale of the drifts (see e.g. https://wiki.kek.jp/display/~gpuglisi/Gain+Drifts+Systematics). Now the drift fluctuates at scales comparable to the observation one (frequencies larger than the observation period are not accounted).
  2. Reducing the jumps between two neighbour gain estimation intervals(see image below). I use a moving window, including a partial overlap (1 min ) between two neighbour intervals of 10 min.
  3. Improving the regression. Before I simply estimated the linear regression coefficient between the template and the drifted TOD(which corresponds to 0th order poly), now I estimate also the coefficients for the 1st and 2nd order polynomial as if follows:
$$ d_{A} = g_A(t) (s_{T} + s_{P} )+ n \\ T =s_{template} \\ f(t, \alpha ) = \sum _i \alpha_i \mathcal{P_L^{(i)}} (t) \\ \hat{\alpha} = LinRegress (f, T, d_{A}) \\ \hat{G}(t) = \sum _i \hat{\alpha}_i \mathcal{P_L^{(i)}} (t) \\ \tilde{d} = d/\hat{G} $$

I tested the mitigation for different choices of templates on 12 h of LB observations and a common set of parameters :

  • fknee_drift= 50 mHz
  • gain fluctuation amplitude = 1e-2 (wich should corresponds to 10 uK, to be checked with Dominic)

Simulation Pipeline :

  1. generate TODs from scanning the input signal and template maps
  2. inject gain drift to the signal TOD
  3. coadd instrumental noise to the signal TOD
  4. Mitigate gain drift with CMB T by HPF filtering the signal +noise TOD, estimate the gain, mitigated the unfiltered TOD.
  5. Run the destriper, etc..
